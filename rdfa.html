RDFa Vocabulary for Term and Prefix Assignment

This document describes the RDFa Vocabulary for Term and Prefix Assignment. The Vocabulary is used to modify RDFa 1.1 processing behavior. The URI of the vocabulary is http://www.w3.org/ns/rdfa#. Turtle and an RDF/XML versions of the vocabulary are also available.

For a precise definition of these vocabulary elements, see the RDFa Core 1.1 specification.

Term and prefix assignment vocabulary

This vocabulary contains terms to define prefix mappings and term assignments; these can be used as part of the @profile mechanism of RDFa 1.1.

Classes

  • rdfa:PrefixMapping: used in conjunction with the definition of prefixes as the domain of the rdfa:prefix property.
  • rdfa:TermMapping: used in conjunction with the definition of terms as the domain of the rdfa:term property.

Both of these classes are subclasses of a common rdfa:VocabularyMapping. The two classes are disjoint.

Properties

  • rdfa:prefix: defines a prefix. The value is supposed to be a NMTOKEN; the property is used on a rdfa:PrefixMapping instance. It is a functional property.
  • rdfa:term: defines a term. The value is supposed to be a NMTOKEN; the property is used on a rdfa:TermMapping instance. It is a functional property.
  • rdfa:uri: defines a uri string to be used either with a term or a prefix definition. The value is supposed to be a valid URI; the property is used on a rdfa:VocabularyMapping instance. It is a functional property.
